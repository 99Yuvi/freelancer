1
Pre-deployment — gather everything
Get all accounts, keys, and configs ready before touching the server
~30 min
CRITICAL
Razorpay live account + live keys
Login to razorpay.com/dashboard → Settings → API Keys → Generate Live Keys. Note down: rzp_live_xxxx key ID and key secret. Also note the Webhook secret (create a webhook in Razorpay pointing to your domain).
CRITICAL
Domain DNS configured
In Hostinger hPanel → Domains → DNS Zone:
• Add A record: @ → Hostinger server IP
• Add A record: www → same IP
• Add A record: chat → DigitalOcean droplet IP
DNS propagation takes 5–30 minutes.
CRITICAL
Email SMTP credentials ready
Option A — Hostinger SMTP: hPanel → Emails → Create mailbox (noreply@operalyn.com) → note SMTP host + port.
Option B — SendGrid: sendgrid.com → Free tier → Settings → API Keys → create one.
IMPORTANT
Production .env file prepared locally
Copy the dev .env and update these values for production:
# Change these for production: APP_ENV=production APP_DEBUG=false APP_URL=https://operalyn.com DB_DATABASE=operalyn_production DB_USERNAME=your_hostinger_db_user DB_PASSWORD=your_db_password MAIL_MAILER=smtp MAIL_HOST=smtp.hostinger.com MAIL_PORT=587 MAIL_USERNAME=noreply@operalyn.com MAIL_PASSWORD=your_email_password RAZORPAY_KEY_ID=rzp_live_xxxx RAZORPAY_KEY_SECRET=xxxx RAZORPAY_WEBHOOK_SECRET=xxxx SANCTUM_STATEFUL_DOMAINS=operalyn.com,www.operalyn.com SESSION_DOMAIN=.operalyn.com FRONTEND_URL=https://operalyn.com CHAT_SERVER_URL=https://chat.operalyn.com NODE_SERVICE_TOKEN=generate_after_first_migrate
OPTIONAL
GitHub repo created and code pushed
Push both projects to GitHub (private repo). This enables git-based deployment. If not using git deploy, you can use SFTP/SCP instead.
2
Hostinger Setup
One-time server configuration via hPanel
~30 min
CRITICAL
Set PHP version to 8.3
hPanel → Advanced → PHP Configuration → select PHP 8.3. Also enable these extensions: pdo_mysql, mbstring, openssl, tokenizer, xml, ctype, json, bcmath, fileinfo, gd
CRITICAL
Create MySQL database + user
hPanel → Databases → MySQL Databases → Create:
• Database name: operalyn_production
• User: create a new user with a strong password
• Grant ALL PRIVILEGES on the database to the user
Note down: hostname (usually localhost), database name, username, password.
CRITICAL
Set document root to /public
hPanel → Websites → Manage → General → Document Root → change from public_html to public_html/public. This points the web server to Laravel's public folder.
CRITICAL
Enable SSL for operalyn.com
hPanel → SSL → Install Let's Encrypt for operalyn.com and www.operalyn.com. Enable "Force HTTPS" option. This may take 5 minutes.
IMPORTANT
SSH access setup
hPanel → Advanced → SSH Access → Generate Key Pair or upload your public key. Test: ssh username@operalyn.com. This is needed for deployment and running artisan commands.
OPTIONAL
Set up git bare repo for push-to-deploy
SSH into server and run:
mkdir -p ~/repos/operalyn.git cd ~/repos/operalyn.git git init --bare # Create post-receive hook cat > hooks/post-receive << 'EOF' #!/bin/bash git --work-tree=/home/USER/public_html --git-dir=/home/USER/repos/operalyn.git checkout -f main cd /home/USER/public_html composer install --no-dev --optimize-autoloader --no-interaction php artisan migrate --force php artisan config:cache && php artisan route:cache && php artisan view:cache chmod -R 775 storage bootstrap/cache echo "Deployed at $(date)" EOF chmod +x hooks/post-receive
Then locally: git remote add production ssh://USER@operalyn.com/~/repos/operalyn.git
3
Backend Deployment
Upload Laravel, run migrations, configure cron
~45 min
CRITICAL
Upload backend files to public_html
Option A (git): git push production main — post-receive hook runs automatically.
Option B (FTP): Upload all files from E:\operalyn-backend to public_html/ via Hostinger File Manager or FileZilla. Do NOT upload the .env file via FTP — create it manually on server.
CRITICAL
Create .env file on server
SSH into server → nano ~/public_html/.env → paste the production .env you prepared in Phase 1. Never use FTP for .env — it may be logged in transit.
ssh username@operalyn.com cd ~/public_html nano .env # Paste your production .env content → Ctrl+O → Enter → Ctrl+X
CRITICAL
Run composer install (production)
cd ~/public_html composer install --no-dev --optimize-autoloader --no-interaction
This installs PHP packages without development dependencies. Takes 2–3 minutes.
CRITICAL
Generate app key + run migrations + seed
php artisan key:generate php artisan migrate --force php artisan db:seed # This creates admin user, 7 categories, 31 skills, 5 settings
CRITICAL
Create storage symlink + set permissions
php artisan storage:link chmod -R 775 storage bootstrap/cache # Verify private storage is OUTSIDE public_html: ls -la storage/app/private/ # should exist
CRITICAL
Warm all production caches
php artisan config:cache php artisan route:cache php artisan view:cache php artisan event:cache
This pre-compiles config, routes, and Blade templates. Required for production performance.
IMPORTANT
Generate NODE_SERVICE_TOKEN
php artisan tinker # In tinker, run: \App\Models\User::find(1)->createToken('node-service')->plainTextToken; # Copy the printed token — needed for chat server .env exit
CRITICAL
Set up cron job in hPanel
hPanel → Advanced → Cron Jobs → Add cron job:
Schedule: Every minute (* * * * *)
Command:
/usr/local/bin/php /home/USERNAME/public_html/artisan schedule:run >> /dev/null 2>&1
This single cron drives the queue worker (every minute), review expiry (daily 2am), and cleanup jobs.
IMPORTANT
Register Razorpay webhook URL
Razorpay Dashboard → Settings → Webhooks → Add new webhook:
• URL: https://operalyn.com/api/v1/webhooks/razorpay
• Events: check payment.captured and payment.failed
• Set webhook secret → copy it to your .env RAZORPAY_WEBHOOK_SECRET
CRITICAL
Verify API is responding
curl https://operalyn.com/api/v1/categories # Should return JSON with 7 categories # {"data":[{"id":1,"name":"Development",...}]}
If 500 error: check storage/logs/laravel.log for details. Common cause: wrong DB credentials or missing .env values.
4
Frontend Deployment
Build React app and upload to server
~15 min
CRITICAL
Update Vite config for production API URL
In E:\operalyn-frontend\vite.config.js the proxy is for dev only — in production the React app and Laravel API are on the same domain, so no proxy config needed. The /api calls automatically go to the same server.
ℹ️ The frontend uses relative paths like /api/v1/ — these work because the React build is served from the same domain as the Laravel API.
CRITICAL
Set VITE_CHAT_URL environment variable
Create E:\operalyn-frontend\.env.production:
VITE_CHAT_URL=https://chat.operalyn.com
This tells the frontend where to connect for WebSocket. Without this, it defaults to localhost:3001 which won't work in production.
CRITICAL
Build React for production
cd E:\operalyn-frontend npm run build # Output: dist/ folder (~580KB JS bundle) # ✓ built in ~500ms — look for this confirmation
CRITICAL
Upload dist/ contents to server
Upload everything inside dist/ to public_html/public/app/ on Hostinger. After upload, the structure should be:
public_html/public/app/ index.html assets/ index-xxxx.js index-xxxx.css
# Using rsync (if SSH available): rsync -avz --delete E:/operalyn-frontend/dist/ username@operalyn.com:~/public_html/public/app/
CRITICAL
Verify .htaccess handles SPA routing
The public/.htaccess (already in the codebase) handles SPA routes. Verify it's uploaded. Test by opening https://operalyn.com/auth/login directly — should load the React app, not a 404.
IMPORTANT
Test landing page and login
Open https://operalyn.com → should show Landing page. Click "Get started" → Register page. Login with admin credentials. Verify dashboard loads with no API errors in browser DevTools (F12 → Console).
5
Chat Server (DigitalOcean)
Node.js + Socket.io on a $6/mo Ubuntu droplet
~45 min
CRITICAL
Create DigitalOcean droplet (Ubuntu 22.04)
DigitalOcean → Create Droplet → Ubuntu 22.04 → Basic → Regular → $6/mo (1vCPU, 1GB RAM). Add your SSH key. Note the droplet IP address.
CRITICAL
Install Node.js + Nginx + PM2
ssh root@YOUR_DROPLET_IP apt update && apt upgrade -y # Install Node.js 20: curl -fsSL https://deb.nodesource.com/setup_20.x | bash - apt install -y nodejs nginx # Install PM2 (process manager): npm install -g pm2 # Verify: node --version # should show v20.x pm2 --version
CRITICAL
Upload chat server files + install dependencies
# Option A — rsync from local: rsync -avz E:/operalyn-chat/ root@DROPLET_IP:/var/www/chat/ # On server: cd /var/www/chat npm install --production cp .env.example .env nano .env # fill in values
Set in .env: LARAVEL_API_URL=https://operalyn.com/api/v1, NODE_SERVICE_TOKEN= (paste from Phase 3 step 7), CORS_ORIGIN=https://operalyn.com
CRITICAL
Start with PM2 + enable auto-start
cd /var/www/chat pm2 start server.js --name "operalyn-chat" pm2 startup # copy and run the printed command pm2 save # Verify it's running: pm2 status # should show "online" pm2 logs operalyn-chat --lines 20 # Should see: "Operalyn chat server running on port 3001"
CRITICAL
Configure Nginx as WebSocket proxy
nano /etc/nginx/sites-available/chat.operalyn.com
Paste this config:
server { listen 80; server_name chat.operalyn.com; location / { proxy_pass http://localhost:3001; proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection "upgrade"; proxy_set_header Host $host; proxy_read_timeout 86400; } }
ln -s /etc/nginx/sites-available/chat.operalyn.com /etc/nginx/sites-enabled/ nginx -t systemctl reload nginx
CRITICAL
SSL for chat.operalyn.com
apt install -y certbot python3-certbot-nginx certbot --nginx -d chat.operalyn.com # Enter email, agree to terms, choose redirect HTTP to HTTPS # Verify auto-renewal: certbot renew --dry-run
6
Verify Everything & Launch
Final checks before going public
~30 min
CRITICAL
APP_DEBUG=false confirmed
SSH → grep APP_DEBUG ~/public_html/.env → must show APP_DEBUG=false. If true, stack traces are exposed to the public.
CRITICAL
Change admin password immediately
Login at https://operalyn.com/auth/login with admin@operalyn.com / Admin@123456 → change password to something strong immediately. Or via artisan:
php artisan tinker \App\Models\User::find(1)->update(['password' => bcrypt('YOUR_NEW_STRONG_PASSWORD')]);
CRITICAL
Test full registration flow on production
On your phone or a different device (not localhost):
1. Open https://operalyn.com
2. Register as a client with a real email
3. Verify you receive the verification email
4. Click link → verify email works
5. Login → dashboard loads correctly
CRITICAL
Test Razorpay payment with live keys (small amount)
Create a test contract → add a ₹1 milestone → approve → complete payment with a real card.
Check: webhook received → payment.captured event → milestone marked paid → invoice generated.
⚠️ Use a minimal amount (₹1) for the first live payment test. The commission will be ₹0.12. Both amounts go through the real payment flow.
CRITICAL
Test real-time chat on production
Open chat in two different devices/browsers on the production URL. Send a message — it should appear in real-time without refresh. Check browser console for any WebSocket errors.
IMPORTANT
Verify private files are not publicly accessible
# Try accessing a private file directly — should get 403 or 404: curl https://operalyn.com/storage/app/private/ # Should NOT return file listing curl https://operalyn.com/../storage/app/private/ # Should return 403 Forbidden
IMPORTANT
Verify cron job is running
php artisan schedule:list # Should show: queue:work, expire-reviews, prune-failed, sanctum:prune-expired # Manually trigger queue to check it works: php artisan queue:work --stop-when-empty # Should process any pending jobs and exit
IMPORTANT
Confirm HTTPS everywhere
Check: https://operalyn.com ✅, https://chat.operalyn.com ✅, no mixed-content warnings in browser console. Try http://operalyn.com → should redirect to HTTPS.
IMPORTANT
Set up backup (Hostinger)
hPanel → Backups → enable automatic weekly backups. Also manually export the database now: hPanel → Databases → phpMyAdmin → Export → SQL format. Store this backup offline.
OPTIONAL
Add FULLTEXT index on projects (MySQL production)
The FULLTEXT index was removed for SQLite dev compatibility. On MySQL production, add it for better search performance:
php artisan tinker DB::statement('ALTER TABLE projects ADD FULLTEXT ft_search (title, description)');
